## אופן השימוש

השתמשנו בעוזר AI לאורך פיתוח המטלה כ**כלי עזר** לביצוע משימות טכניות. התהליך היה שיחה דו-כיוונית: אנחנו הגדרנו את הדרישות, קיבלנו הסברים, שאלנו שאלות, ואישרנו כל שינוי.

### מה ביקשנו מה-AI

| תחום | השימוש |
|------|--------|
| **כתיבת קוד** | יצירת קוד ל-Pipeline, VIF, PPS, Error Analysis — לפי הנחיות שנתנו |
| **תיקון שגיאות** | איתור וניפוי באגים (קריסת RF, שגיאות KeyError, חוסר יציבות VIF) |
| **Git ו-GitHub** | ניהול ה-commits, push לענפים `master` ו-`reuven`, כתיבת הודעות commit ברורות |
| **בניית פיצ'רים** | הבנת אילו פיצ'רים נכון לבנות — מה מוסיף מידע, מה יוצר Data Leakage, כיצד לחשב LOO Bayesian encoding, prestige_count, negative_count וכו' |
| **עיצוב המחברת** | פורמט HTML, טבלאות Markdown, עיצוב ויזואלי |
| **תיעוד** | כתיבת הסברים לתאי Markdown בהתאם לחומרי הקורס |

### מה עשינו בעצמנו

- **קריאת חומרי הקורס** — עברנו על כל המצגות והתמלולים וזיהינו אילו נושאים נדרשים
- **הבנת הנתונים** — ניתחנו את dataset.csv, זיהינו בעיות missing values, הכרנו את המאפיינים
- **קבלת ההחלטות** — בחרנו אילו פיצ'רים לכלול, מה VIF מקובל, מה להסיר ולמה
- **אימות התוצאות** — הרצנו את המחברת, בדקנו שהתוצאות הגיוניות, דיווחנו על שגיאות
- **הכוונה תמידית** — בכל שלב הגדרנו מה אנחנו רוצים והאם הפלט תואם את מה שלמדנו

## דוגמאות לשיתוף פעולה

**Git ו-GitHub** — לא ידענו כיצד לנהל כמה ענפים ולוודא שהם מסונכרנים. ה-AI הסביר את הפקודות הנכונות ודחף כל שינוי לשני הענפים בו-זמנית.

**בניית פיצ'רים** — שאלנו "אילו פיצ'רים יגרמו למודל להשתפר?" וה-AI הסביר את הרציונל מאחורי כל פיצ'ר: מדוע LOO מונע leakage, מדוע `prestige_count` חזק מ-`genre_Documentary` בודד (r=+0.282 לעומת אינדיקטור בינארי), ומדוע `known_actor_ratio` יוצר VIF גבוה.

**תיקון שגיאות** — כאשר הקומפיוטר קרס בהרצת Random Forest, לא ידענו מדוע. ה-AI זיהה את הבעיה (`n_jobs=-1` מקוננים) והסביר את הפתרון לעומק.

## עמדתנו

השימוש ב-AI עזר לנו להתמקד בהבנה הרעיונית ובניתוח — במקום לבזבז זמן על syntax ו-debugging. כל רכיב במחברת נבדק ומובן על-ידינו. אנו עומדים מאחורי כל ההחלטות שהתקבלו לגבי בחירת הפיצ'רים, המודלים והניתוחים.
